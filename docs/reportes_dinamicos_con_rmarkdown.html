## Crisis de reproducibilidad en la ciencia

La mayor√≠a de los procedimientos (incluyendo an√°lisis de datos) en estudios cient√≠ficos no se pueden replicar

```{r, echo=FALSE,out.width="70%", fig.align='center'}


knitr::include_graphics("./images/nature-reproducibility-crisis.jpeg")

```

[*Baker. 2016. Nature*](https://www.nature.com/articles/533452a)

 
El compartir datos y an√°lisis de forma transparente y detalladamente documentada, en formatos que puedan ser reproducidos por otros investigadores es una de las principales herramientas para lidiar con este problema.  

```{r, echo=FALSE,out.width="100%", fig.align='center'}

knitr::include_graphics("./images/soluciones_reproducible.png")

```

*Gallagher et al. 2020. Nature Eco & Evo* 

Las herramientas program√°ticas como R tienden naturalmente a facilitar la reproducibilidad ya que el c√≥digo de un an√°lisis se puede registrar y compartir f√°cilmente. Muchos lenguajes de programaci√≥n permiten la generaci√≥n de reportes din√°micos, los cuales resultan fundamentales para hacer manejo de datos y an√°lisis estad√≠sticos reproducibles. Los reportes din√°micos suelen contener toda la informaci√≥n necesaria para que otros investigadores puedan replicar todos los pasos de an√°lisis que generaron los resultados de art√≠culos cient√≠ficos. Por tanto son una herramienta indispensable para mejorar la reproduciblidad en la ciencia.

En R la herramienta mas popular para la generaci√≥n de reportes din√°micos es Rmarkdown. Rmarkdown [se puede describir](http://rmarkdown.rstudio.com) como "*un formato el√©ctronico de documentaci√≥n que facilita la creaci√≥n de documentos, presentaciones e informes din√°micos desde R*". R Markdown puede generar documentos con bloques de c√≥digo de R ("chunks") incrustados entre l√≠neas de texto. Cuando se procesa el documento, estos bloques se eval√∫an (si el usuario as√≠ lo desea) y los resultados se "imprimen" en el documento de acuerdo con ciertas convenciones.

&nbsp;

### Ventajas de los reportes din√°micos con Rmarkdown:

- El **c√≥digo R se puede incrustar en el informe**, por lo que no es necesario mantener el informe y el c√≥digo de R por separado

- Incluir el c√≥digo R directamente en un informe proporciona una **estructura intuitiva** para reproducir los an√°lisis

- El texto del informe est√° **escrito como texto normal**, por lo que no se requieren conocimientos de programaci√≥n (i.e. R o HTML) para comprenderlos

- **El resultado es un archivo HTML** que incluye im√°genes, bloques de c√≥digo con los comandos de R, los resultados de estos c√≥digos y texto. No se necesitan archivos adicionales, todo est√° incorporado en el archivo HTML.

- Los informes son **f√°ciles de compartir por correo o publicarlos en l√≠nea**

- Estos informes **facilitan la colaboraci√≥n y mejoran la reproducibilidad** (entender los an√°lisis es mucho mas f√°cil cuando hay texto explicativo, c√≥digo de R, los resultados del c√≥digo y los gr√°ficos en un mismo archivo)

- **Se actualizan f√°cilmente** para incluir nuevos an√°lisis y/o integrar nuevos datos

## Metadatos (YAML)

Hay tres componentes b√°sicos de un documento de R Markdown: los metadatos, el texto y el c√≥digo. Los metadatos se escriben entre el par de tres guiones ("- - -") generalmente al inicio del documento. La sintaxis de los metadatos es [YAML (YAML Ain‚Äôt Markup Language)](https://en.wikipedia.org/wiki/YAML), por lo que a veces tambi√©n se denomina metadatos YAML. La sangr√≠a es importante en YAML, as√≠ que debe a√±adirla a los subcampos (con respecto a un campo superior).

Este encabezado muestra las opciones mas com√∫nmente usadas en los metadatos YAML:

```{r, eval = FALSE}

    fig_caption: yes

    toc: yes
    toc_float: yes
    df_print: paged
---

```

  titlePanel("Ejemplo"),
  
  sidebarLayout(
    sidebarPanel(
      sliderInput(inputId = "bins",
                  label = "Numero de barras:",
                  min = 1,
                  max = 50,

    )
  )

server <- function(input, output) {
 
  output$distPlot <- renderPlot({
    
    x    <- faithful$waiting
    bins <- seq(min(x), max(x), length.out = input$bins + 1)
    
    hist(x, breaks = bins, col = "#3E4A89FF", border = "white",
         xlab = "Tiempo de espera para la siguiente erupcion",
         main = "Histograma del tiempo de espera")
  })
}
